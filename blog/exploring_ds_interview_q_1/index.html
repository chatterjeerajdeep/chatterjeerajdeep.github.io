Exploring Data Science Interview Questions - Day 1

4 interesting questions from Microsoft data science interviews

Published on Jul 07, 2022

Reading time: 4 minutes.

Question 1

Generate a fair coin from a biased one

If there is a biased coin, it means that the probability of H and T are not equal. Let’s say that the probability for H is p. That of T would be q = (1-p).

If we flip the biased coin 2 times, then following are the possibilities and their corresponding probabilities:
P(HH) = p2
P(HT) = pq
P(TH) = qp
P(TT) = q2

We notice that the P(HT) and P(TH) are same. So, if we flip a coin two times and get the same result (HH or TT) we discard the trial and start fresh. But if we get opposite results (HT or TH), then we choose the first result.

This is a time taking process. The more biased the coin is, the more time taking it is to get fair results. This idea was suggested by John Von Neumann. Here’s the link and here’s a simpler explanation

Question 2

Generate 7 integers with equal probability from a function which returns 1/0 with probability p and (1-p)

Let’s safely assume that p is neither 0 nor 1.

This is an extension of the previous question. Let’s think about solving this with the help of a fair coin. If we wanted to generate 7 integers with equal probability by tossing a fair coin, we could have tossed the coin 3 times. Why? Because by tossing a coin 3 times, we can generate 23=8 different outcomes. Each of these 8 outcomes would have equal probabilities.
HHH -> assign 1
HHT -> assign 2
HTH -> assign 3
HTT -> assign 4
THH -> assign 5
THT -> assign 6
TTH -> assign 7
TTT -> ignore

Now, for a biased coin. I found 3 interesting solutions here.

  1. As the answer to the previous question suggests, we can generate a fair coin from a biased coin, by tossing the coin 2 times and then selecting the outcomes HT as H and TH as T.

So our table should now look like this:

Fair Coin Biased Coin Assign
HHH HT HT HT 1
HHT HT HT TH 2

and so on..

This solution definitely works, but it generates a lot of wasted trials.

  1. The second approach suggested is to toss the unfair coin 5 times. There can be 10 ways to get 3 H and 2 T (3C2 = 10). All of them have equal probabilities (p3q2). So we can take any 7 of them, assign a number from 1-7 to each and discard the rest.

I took some time to think about why 5 tosses, why not 4? I realized, by tossing 4 coins, there can not be a situation where I get the same probability for 7 or more outcomes.

  1. The third approach suggested is to toss the unfair coin 7 times. It is kind of similar to the previous approach, but since we are tossing the coin 7 times, we are generating too many combinations. We get 7 different arrangements with 1 H, 21 with 2 H, 35 with 3 H, 35 with 4 H, 21 with 5 H, 7 with 6 H and 1 with 7 H. We can simply count the number of H in a combination and assign it a number.

Question 3

How do you detect if a new observation is an outlier?

There are several methods to detect outliers that are suggested by data science practitioners. Here’s the list of all the methods that I have found -

  1. Scatter plot
  2. Box plot
  3. Standard Deviation Method
  4. IQR method
  5. Hypothesis testing like:
  • Grubb’s test
  • chi-square test
  • Dixon’s q test
  1. Isolation forest
  2. Robust Random Cut Forest
  3. Sorting the data

The majority of data science practitioners believe that the visual methods (like box plots and scatter plots, histograms) and simple sorting of the data are sometimes much more effetive in detecting outliers than statistical or hypothesis testing.

Question 4

Find the maximum of subsequence in an integer list.

This question was not very clear. I found several related questions in leetcode. Here’s the links of the questions and my solutions:

Question - https://leetcode.com/problems/find-subsequence-of-length-k-with-the-largest-sum/

Solution -

import heapq
class Solution:
    def maxSubsequence(self, nums: List[int], k: int) -> List[int]:
        heap_list = []
        # for every number and its index in the list
        for i,num in enumerate(nums):
            # push a tuple of number and index into the heap
            # since this is a min heap, the least num is at the top
            if len(heap_list) < k: 
                # push data into heap as long as length is less than k
                heapq.heappush(heap_list, (num, i))
            else:
                # when length becomes greater than k, push the data and
                # pop the smallest entry
                heapq.heappushpop(heap_list, (num, i))
        
        return [tup[0] for tup in sorted(heap_list, key=lambda tupl:tupl[1])]

Reference - this link is not the best explanation out there, but it lead me to study and understand how to use heap for such questions. I also got the know about the heapq python package
