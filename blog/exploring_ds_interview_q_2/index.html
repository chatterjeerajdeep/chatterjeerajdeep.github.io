5 interesting questions from Microsoft data science interviews

Question 1 - Can you explain the difference between a Validation Set and a Test Set?

What is a good model in machine learning?

A model is good if it can generalize on the unseen data. When a model is trained on some data, the model is expected to perform well on that data. But in use, the model has to predict on unknown data. If a model fails to generalize, then the predictions of the model may not be of much use.

How can we know if a model will generalize well?

We split our entire data set into 2 sets - train and test. We perform the process of training using the train set. We then use the test set to evaluate the model’s performance. We can have different models, with different hyper-parameters and see which model works best for the test set.

Since, the test set is unseen for the model, we can get a sense of which model performs well on the unseen data.

What is the issue with this approach?

We are choosing the best model based on the test set performance. This makes the test set ineligible for evaluating how well the chosen model generalizes on unseen data. The test set may not be fully representative of the overall variations in the actual unseen data and our chosen model might just happen to work well for the given test set.

What is the solution?

We split our data into 3 sets - train, validation and test. The train set is used to train the model. The validation set is used to choose the best performing model. The test set is used to evaluate the best performing model on completely unseen data.

If the model that performs the best on the validation set, also performs well on the test set, then it’s a model which can generalize well.

But if the model performance on the validation set and the test set are very different, then we have to start all over again.

Question 2 - Explain Cross Validation

What is the motivation?

When we have limited amount of data, it becomes difficult to evaluate the model. A simple train/validation split creates a tradeoff between the train and the validation set, because the data used for training can not be used for validation.

Wikipedia says, Assume a model with one or more unknown parameters, and a data set to which the model can be fit (the training data set). The fitting process optimizes the model parameters to make the model fit the training data as well as possible. If an independent sample of validation data is taken from the same population as the training data, it will generally turn out that the model does not fit the validation data as well as it fits the training data. The size of this difference is likely to be large especially when the size of the training data set is small, or when the number of parameters in the model is large.

How can cross-validation help?

Cross-validation is a resampling method that systematically creates and evaluates multiple models on multiple subsets of the dataset. Instead of sticking with a single train/validation split, we can use the multiple subsets of the dataset for training and validation.

These are the steps followed:

  1. Shuffle the dataset randomly.
  2. Split the dataset into k groups For each unique group: Take the group as a hold out or validation data set Take the remaining groups as a training data set
  3. Fit a model on the training set and evaluate it on the validation set
  4. Retain the evaluation score
  5. Summarize the skill of the model using the sample of model evaluation scores

To quote from this resource: Importantly, each observation in the data sample is assigned to an individual group and stays in that group for the duration of the procedure. This means that each sample is given the opportunity to be used in the hold out set 1 time and used to train the model k-1 times.

Summarizing the advantages and disadvantages of Cross-Validation:

Advantages:

  1. More efficient use of data - We can use a limited sample in order to estimate how the model is expected to perform in general when used to make predictions on data not used during the training of the model.

  2. We repeat the model evaluation process multiple times (instead of one time) and calculate the mean skill. The mean estimate of any parameter is less biased than a one-shot estimate.

3.This resource suggests that the purpose of cross-validation is to identify how well a setting can generalize, and once we are satisfied, we can use the entire data to train our final model.

But retraining a model may be time taking. We can make use of the models built during cross validation and use them in ensemble. This will help us reduce variance.

Question 4 - What is Cluster Sampling

Cluster Sampling is one of the sampling strategies where we look at the naturally occuring groups in the population and then randomly choose a group to examine all the samples in that group.

This is logically a practical approach of sampling.

For example, we want to know the quality of items manufactured by a company. There are millions of items in the market and it is not possible to pick up each item and test for the quality.

But we know that these items are being manufactured by 4 factories in the country. This is a natural grouping. We now choose any factory at random, and examine each item manufactured by that factory in a given day. This is cluster sampling.

This will work, only when all the factories have the same status of their machineries. However, if one factory is older or not as well maintained as others, this strategy will create biased samples.

Question 5 - What is Systematic Sampling

Systematic Sampling is one of the sampling strategies where we define a system to extract our samples.

For example, we want to test the quality of items manufactured in a factory. Now in a day a factory produces lots of items. Examining each one of them may be inconvenient. So we define a system - we pick up every 10th item produced and examine that.

The sample thus generated using this strategy may generate biased samples, if there is a pattern in the population. For example, if the machines generate items in a sequence, then every 10th item may not be representative of all other items.
